  1. Description of ajax calls

    • Call to github to get user information
      1. "https://api.github.com/users/user"
      2. user, it's the value of the input.
      3. the response is a big json that containing the user details and we take from it the keys:
        1. login
        2. public_repos
        3. repos_url
        4. followers
        5. followers_url
    • Call to github to get user repositories
      1. "https://api.github.com/users/user/repos"
      2. user/repos, it came from repos_url key.
      3. The response is a json that containing all the public repositories of the user, we take from it the keys:
        1. full_name
        2. html_url
    • Call to github to get user followers
      1. "https://api.github.com/users/user/followers"
      2. user/followers, it came from followers_url key.
      3. The response is a json that containing all the followers of the user, we take from it the keys:
        1. login
        2. html_url
    • Call to are server to save user
      1. "http://localhost:3000/save?username='user'"
      2. username, it's the value of the input.
      3. The response is a json that containing the details of the request, the keys are:
        1. status
        2. details
    • Call to are server to delete user
      1. "http://localhost:3000/delete?username='user'"
      2. username, it's the value of the input.
      3. The response is a json that containing the details of the request, the key is:
        1. status
    • The server call to github to make sure that we will save exist github user
      1. "https://api.github.com/users/user"
      2. user, it's came from the request.
      3. the response is a big json that containing the user details and we not take from it nothing, we gust check the status of the request.
    • Call to are server to load data list
      1. "http://localhost:3000/load"
      2. The response is a json that containing the details of the request, the keys are:
        1. status
        2. list

    The login system allow to detect on the sensitive data.
    When the user call to "localhost:3000" and he never login before so he move lo login page.
    When the user try to login and he not register he not move to the search page and he need to register and login.
    When the user try to login and he register he move to the search page and cookie create to stay him connect.
